# Desktop Workflows

Ниже не справочник, а простые живые сценарии.

## Сценарий 1. Начать новую задачу

1. Открой экран `Проект`
2. Напиши задачу простыми словами
3. Нажми `Сделать план`
4. Если план выглядит нормально, переходи в `Задачи`
5. Запускай `Безопасный запуск` или `Реальный запуск`

Это основной путь для большинства людей.

## Сценарий 2. Посмотреть результат

После запуска открой `Задачи`.

Там можно:

- увидеть статус
- открыть изменённые файлы
- посмотреть отчёты
- перейти в чат

## Сценарий 3. Обсудить задачу с агентом

Если задача уже запускалась:

1. открой её на экране `Задачи`
2. нажми `Обсудить в чате`
3. продолжай разговор уже с учётом результата

Так чат остаётся связанным с реальной работой, а не превращается в отдельную болталку.

## Сценарий 4. Использовать файл как дополнительный контекст

Если нужно:

- посмотреть diff
- открыть конкретный файл
- вручную поправить текст

тогда переходи на экран `Файлы`.

После этого файл можно:

- добавить в новую задачу
- добавить в чат

## Сценарий 5. Поставить готовый набор

1. Открой `Наборы`
2. Выбери нужный набор
3. Сначала посмотри preview
4. Если всё нормально, нажми `Установить`

Если есть конфликты, сначала лучше обсудить их через `Чат` или новую задачу.

## Сценарий 6. Вернуться к незавершённой работе

Если вчера работа оборвалась:

1. открой `Задачи`
2. выбери последнюю задачу
3. посмотри изменённые файлы и отчёты
4. продолжи через `Чат` или через новую задачу

Это один из главных плюсов ARC: не нужно собирать контекст заново.
