# CLI для новичков

Этот гайд для человека, который хочет пользоваться ARC из терминала, но без изучения всей внутренней архитектуры.

## Как думать про CLI

Простой способ:

- `arc doctor` — проверить, что всё готово
- `arc init` — подготовить проект
- `arc task plan` — сначала сделать план
- `arc task run` — потом сделать запуск
- `arc run status` — посмотреть, что получилось

## Шаг 1. Подготовить проект

```bash
./bin/arc doctor
./bin/arc init --path .
./bin/arc mode set work --path .
```

## Шаг 2. Сделать первый план

```bash
./bin/arc task plan --path . --mode work --provider codex "Посмотри проект и предложи следующее безопасное изменение"
```

После этого ARC создаст run и артефакты в `.arc/runs/...`.

## Шаг 3. Сделать безопасный запуск

```bash
./bin/arc task run --path . --mode work --provider codex --dry-run "Подготовь небольшое изменение и опиши, что будет сделано"
```

Безопасный запуск полезен, когда ты хочешь сохранить весь workflow, но пока не отдавать реальный live-запуск агенту.

## Шаг 4. Посмотреть результат

```bash
./bin/arc run list --path .
./bin/arc run status --path .
./bin/arc questions show --path .
```

Что ты обычно увидишь:

- статус задачи
- список артефактов
- открытые вопросы

## Шаг 5. Продолжить работу через чат

```bash
./bin/arc chat start --path . --provider codex --mode work "Обсуди со мной следующий шаг по этой задаче"
```

Если чат уже существует:

```bash
./bin/arc chat list --path .
./bin/arc chat send --path . --session-id <id> "Продолжим разговор"
```

## Шаг 6. Обновить карты проекта

```bash
./bin/arc docs generate --path . --apply
```

## Когда использовать CLI

CLI особенно удобен, если:

- ты любишь терминал
- хочешь полный контроль
- пишешь скрипты
- работаешь удалённо или по SSH
