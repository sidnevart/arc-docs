# ARC

`ARC` помогает работать с AI над проектом без хаоса.

Простая идея:

- проект — это главная точка входа
- задача — это основная единица работы
- чат — это прямой разговор с агентом
- файлы нужны как дополнительный контекст, а не как стартовая точка

Если совсем коротко, ARC нужен чтобы:

- не терять контекст между сессиями
- видеть историю задач и артефактов
- пользоваться и CLI, и desktop без разрыва между ними
- безопасно пробовать изменения до реального запуска

## Что есть в продукте

Сейчас в репозитории уже работают:

- `arc` CLI
- desktop preview
- native desktop app на Wails
- режимы `study`, `work`, `hero`
- локальная память проекта
- run artifacts
- presets
- прямой chat с установленным агентом

## Какой способ выбрать

### Desktop

Выбирай desktop, если тебе нужен простой и понятный интерфейс:

- экран `Проект`
- экран `Задачи`
- экран `Чат`
- экран `Наборы`
- экран `Файлы` только когда он реально нужен

### CLI

Выбирай CLI, если тебе привычнее терминал и нужен точный контроль над командами.

## Что важно знать

ARC local-first:

- проект остаётся на диске
- `.arc/` хранит локальные артефакты и состояние
- desktop не заменяет CLI, а делает его проще для обычного пользователя
