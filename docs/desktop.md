# ARC Desktop

`ARC Desktop` — это простой визуальный слой поверх ARC.

Он не заменяет CLI. Он делает основные действия понятнее:

- посмотреть состояние проекта
- запустить задачу
- продолжить разговор с агентом
- поставить готовый набор
- открыть файл только тогда, когда это действительно нужно

## Два desktop-режима

### 1. Browser preview

Подходит для быстрой проверки и разработки интерфейса.

Запуск:

```bash
make desktop-preview
```

### 2. Native app

Подходит для обычной ежедневной работы.

Launcher лежит в `cmd/arc-desktop-wails`, а сам desktop package — в `apps/desktop/wailsapp`.

## Главная идея интерфейса

В ARC Desktop работа начинается не с файла, а с проекта.

Поэтому основной поток такой:

1. открыть `Проект`
2. сформулировать задачу
3. перейти в `Задачи`
4. при необходимости обсудить результат в `Чате`
5. открыть `Файлы` только если нужно посмотреть diff или править конкретный файл

## Основные разделы

### Проект

Стартовая точка.

Здесь видно:

- имя проекта
- путь к проекту
- режим
- доступные агенты
- последнюю задачу
- основные кнопки запуска

### Задачи

Основной рабочий экран.

Здесь можно:

- сделать план
- сделать безопасный запуск
- сделать реальный запуск
- посмотреть результат
- открыть изменённые файлы
- продолжить работу в чате

### Чат

Прямой разговор с установленным агентом.

Здесь можно:

- начать новый чат
- продолжить старый чат
- отправить контекст вместе с сообщением

### Наборы

Готовые наборы правил и файлов.

Здесь можно:

- посмотреть список наборов
- открыть preview
- увидеть конфликты
- установить набор

### Контекст

Сводка того, что ARC уже знает о проекте.

### Файлы

Дополнительный экран.

Он нужен не для старта работы, а для таких случаев:

- посмотреть конкретный файл
- увидеть diff
- слегка поправить текст
- приложить файл к следующей задаче или чату

## Что такое контекст

В ARC Desktop можно явно прикреплять:

- задачу
- файл
- preview набора

Это лучше, чем неявная “активная вкладка”, потому что пользователь понимает, что именно уйдёт агенту.
